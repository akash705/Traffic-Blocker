Privacy Policy

Traffic Blocker

Effective date: April 14, 2026

Overview

Traffic Blocker is an open-source Android app that blocks internet access on a per-app basis using a local VPN. It is designed with privacy as a core principle: no data leaves your device.

Data Collection

Traffic Blocker does not collect, transmit, or share any personal data. Specifically:

  • No analytics or tracking libraries are included.
  • No crash reporting services are used.
  • No user accounts or registration are required.
  • No advertising SDKs are included.
  • No data is sent to any server operated by us or any third party.

Local VPN

Traffic Blocker uses Android's VpnService API to create a local, on-device VPN tunnel. This VPN:

  • Runs entirely on your device. No traffic is routed to any external server.
  • Only intercepts DNS queries for apps you have selected.
  • Does not inspect, log, or transmit the content of your network traffic.
  • Is scoped to specific apps using Android's per-app VPN feature — unselected apps are unaffected.

DNS Query Log

The app provides an optional DNS query log that shows which domains were queried and whether they were blocked or allowed. This log:

  • Is stored locally on your device only.
  • Is never transmitted off-device.
  • Can be cleared at any time from within the app.

Blocklist Downloads

When you add a blocklist URL, the app downloads the list directly from that URL to your device. During this download:

  • Only the blocklist file is fetched. No user data, device identifiers, or other information is sent.
  • The request is a standard HTTP(S) GET request with no custom headers or tracking parameters.
  • Downloaded blocklists are stored locally on your device.

The blocklist provider's own privacy policy governs how they handle incoming requests to their servers.

DoH/DoT Provider Blocking

To ensure DNS-based blocking is effective, the app blocks connections to known DNS-over-HTTPS (DoH) and DNS-over-TLS (DoT) provider IP addresses for selected apps. This prevents those apps from bypassing DNS blocking via encrypted DNS. No traffic to these IPs is logged or inspected — connections are simply refused.

Permissions

The app requests the following permissions, each necessary for its core functionality:

  • VPN: Creates the local VPN tunnel for DNS interception.
  • Internet: Downloads blocklist files from user-provided URLs.
  • Notifications: Displays the foreground service notification while blocking is active.
  • Usage Stats: Detects which app is in the foreground for background blocking mode.
  • Boot Completed: Restarts blocking automatically after device reboot (if enabled).
  • Query All Packages: Lists installed apps so you can select which ones to block.
  • Battery Optimization Exemption: Keeps the blocking service running reliably.

Data Storage

All app data — settings, selected apps, blocklists, profiles, and DNS logs — is stored locally on your device using standard Android storage mechanisms. No cloud sync or backup to external servers occurs.

Third-Party Services

Traffic Blocker does not integrate with any third-party services, SDKs, or APIs beyond what is described in the Blocklist Downloads section above.

Children's Privacy

Traffic Blocker does not collect any personal information from anyone, including children under the age of 13.

Your Control

You have full control over your data:

  • Clear DNS logs at any time from within the app.
  • Remove blocklists at any time.
  • Uninstalling the app removes all locally stored data.
  • Disconnect the VPN at any time via the app, Quick Settings tile, or Android system settings.

Open Source

Traffic Blocker is open source. You can review the full source code at github.com/akash705/Traffic-Blocker to verify these privacy practices.
