Features

🔒

Per-App Traffic Control

Select which apps to block from a list of all installed apps. Each app can be independently configured.

⚖

Two Blocking Modes

Block All Traffic cuts off internet completely. Block Domains only blocks matching domains from your blocklists.

🌑

Background Blocking

Block an app's network access when it's not in the foreground — stop background ads, telemetry, and data usage.

📜

Custom Blocklists

Add your own blocklist URLs. Supports plain domain lists and hosts-file format. Ships with Hagezi list recommendations.

📈

DNS Query Log

Real-time log of all DNS queries showing blocked vs allowed domains per app.

🗃

Blocking Profiles

Save and switch between different blocking configurations for quick toggling between setups.

⚡

Quick Settings Tile

Start or stop the blocker from the notification shade — no need to open the app.

🔄

Launcher Shortcuts

Long-press the app icon for Start/Stop shortcuts. Compatible with Samsung Routines automation.

How It Works

1

A local VPN tunnel intercepts only DNS traffic — no other data passes through

2

Only selected apps are routed through the tunnel using Android's per-app VPN scoping

3

DNS queries are parsed and matched against your blocklists in real time

4

Blocked queries return 0.0.0.0 — allowed queries are forwarded upstream normally

5

DoH/DoT providers are blocked to prevent DNS bypass via encrypted DNS

Non-DNS traffic is never routed through the VPN. Apps using "Block Domains" mode still access non-blocked sites at full speed.

Requirements

  • VPN permission (prompted on first use)
  • Usage Stats permission (for foreground app detection)
  • Battery optimization exemption recommended for reliable background operation
